## BAGIAN 1 — SECURITY FIXES (Prioritas Tertinggi)

### Fix 1.1 — XSS di Markdown Renderer

**Lokasi:** `src/markdown-render.jsx`

**Masalah:** Fungsi `renderMarkdown()` tidak meng-escape HTML dari input user sebelum apply markdown. Hasil di-render via `dangerouslySetInnerHTML` di kurasah-editor.jsx line 199. Kalau user nulis `<script>` atau `<img onerror=...>` di catatan, itu akan execute.

**Fix:** Tambahkan fungsi escape HTML dan apply di SETIAP tempat yang inject teks user. Edit `src/markdown-render.jsx`:

Tambahkan helper escape di awal fungsi `renderMarkdown`:

```js
const renderMarkdown = (raw) => {
  if (!raw || typeof raw !== "string") return "";

  // ── HTML escape (XSS protection) ──
  const esc = (s) => String(s)
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#39;");

  const lines = raw.split("\n");
  // ... rest sama
```

Lalu di fungsi `inline()`, **escape DULU sebelum apply markdown** (supaya tag markdown kita tetap jalan tapi HTML user di-neutralize):

```js
const inline = (text) => {
  return esc(text)   // ← ESCAPE DULU
    .replace(/\*\*(.+?)\*\*/g, '<strong class="text-ink font-semibold">$1</strong>')
    .replace(/\*(.+?)\*/g, '<em class="text-ink-muted italic">$1</em>')
    .replace(/`(.+?)`/g, '<code class="font-mono text-xs bg-white/8 px-1.5 py-0.5 rounded text-violet-200">$1</code>');
};
```

**Penting:** pastikan SEMUA tempat yang masukin teks user ke HTML output lewat `inline()` atau di-escape manual. Cek di seluruh fungsi `renderMarkdown` — heading, blockquote, list item, paragraf, semua harus lewat `esc()` atau `inline()`. Bagian yang bukan teks user (class CSS, tag HTML yang kita generate sendiri) tidak perlu di-escape.

**Test:** Buat catatan baru di Kurasah dengan isi: `<script>alert('xss')</script>` dan `<img src=x onerror="alert(1)">`. Preview harus menampilkan teks literal `<script>...` sebagai teks biasa, BUKAN execute.

### Fix 1.3 — Payment Modal Auto-Confirm

**Lokasi:** `src/layout.jsx:299-300`

**Masalah:**
```js
window.open("https://lynk.id/madad", "_blank", "noopener,noreferrer");
setTimeout(() => setPaid(true), 800);  // ← auto "paid" 800ms setelah klik
```
User klik tombol bayar → 800ms kemudian modal otomatis berubah ke "Pembayaran sedang diproses" — terlepas apakah user benar-benar bayar. Misleading.

**Fix:** Ganti auto-timeout dengan **konfirmasi manual user**. Edit handler:

```jsx
const handlePayClick = () => {
  window.open("https://lynk.id/talqih", "_blank", "noopener,noreferrer");  // update URL ke talqih juga
  // JANGAN auto-set paid. Tunggu user konfirmasi manual.
};

const handleConfirmPaid = () => {
  setPaid(true);
};
```

UI: setelah user klik "Bayar via Lynk.id", tab Lynk terbuka. Di modal, tampilkan state baru: tombol "Saya sudah bayar" yang user klik **setelah** benar-benar selesai checkout. Baru itu munculkan screen "Pembayaran sedang diproses, kode akan dikirim admin".

```jsx
{!paid ? (
  <div>
    <button onClick={handlePayClick} className="btn-primary w-full py-4">
      Bayar via Lynk.id →
    </button>
    <button 
      onClick={handleConfirmPaid} 
      className="btn-ghost w-full py-3 mt-3 text-sm"
    >
      Saya sudah selesai bayar
    </button>
    <p className="text-xs text-ink-soft text-center mt-3">
      Klik "Bayar via Lynk.id" → selesaikan pembayaran → kembali ke sini dan klik "Saya sudah selesai bayar".
    </p>
  </div>
) : (
  <div className="text-center">
    {/* screen "Pembayaran sedang diproses, kode dikirim admin via WhatsApp" */}
  </div>
)}
```

**Catatan:** ini bukan verifikasi pembayaran real (itu butuh webhook Lynk). Tapi minimal flow-nya jujur — user yang konfirmasi sendiri, bukan timer otomatis yang bohong.

Update juga URL Lynk dari `lynk.id/madad` → `lynk.id/talqih` (atau URL Lynk aktual lo).

### Fix 2.3 — QuickNote Button Universal

**Lokasi:** `src/app.jsx` + hapus dari masing-masing page

**Masalah:** `<QuickNoteButton/>` cuma di-mount di dashboard.jsx, kurasah.jsx, muqaranah-detail.jsx. Di Maddah detail, Tool Guide, Learning Path, Etika — gak ada.

**Fix:** Pindahkan `<QuickNoteButton/>` ke `app.jsx` sebagai sibling dari `<main>`, conditional render untuk member yang sudah onboarded dan bukan di halaman admin:

Di `src/app.jsx`, di dalam layout member (setelah `<main>`):

```jsx
<main className="flex-1">
  {renderPage()}
</main>
{session && profile?.onboarded && <QuickNoteButton/>}
```

Lalu **hapus** `<QuickNoteButton/>` dari:
- `src/pages/dashboard.jsx` (line 190)
- `src/pages/kurasah.jsx` (line 250)
- `src/pages/muqaranah-detail.jsx` (line 270)

**Catatan:** Di muqaranah-detail, QuickNoteButton dipanggil dengan props `contextLabel` dan `contextSource` (untuk auto-link note ke muqaranah). Karena sekarang di-mount global di app.jsx, context-aware linking ini hilang. Solusi: buat QuickNoteButton **detect context dari URL** sendiri:

```jsx
// Di src/quick-note.jsx, dalam QuickNoteButton:
const QuickNoteButton = () => {
  // Auto-detect context dari URL
  const path = window.location.hash.slice(1);
  let contextLabel = null;
  let contextSource = null;
  
  if (path.startsWith("/paths/muqaranah?id=")) {
    const id = new URLSearchParams(path.split("?")[1]).get("id");
    const entry = (typeof getMuqaranahById !== "undefined") ? getMuqaranahById(id) : null;
    if (entry) {
      contextLabel = "Muqaranah: " + entry.title;
      contextSource = { type: "muqaranah", id: entry.id, label: entry.title };
    }
  } else if (path.startsWith("/maddah/")) {
    const id = path.match(/\/maddah\/([\w-]+)/)?.[1];
    const maddah = (typeof getMaddahById !== "undefined") ? getMaddahById(id) : null;
    if (maddah) {
      contextLabel = "Maddah: " + maddah.name;
      contextSource = { type: "maddah", id: maddah.id, label: maddah.name };
    }
  }
  
  // ... rest of component pakai contextLabel & contextSource yang ter-detect
};
```

Ini bikin QuickNote tetap context-aware (auto-link ke Muqaranah atau Maddah yang lagi dibuka) tapi sekarang available di semua page.

## BAGIAN 4 — FINAL ANTI-PATTERN SWEEP

Lakukan pencarian menyeluruh di SEMUA file dan bersihkan:

### 4.1 — Emoji Anti-Pattern

Cari di seluruh `src/`:
- `🎉` `🔥` `🏆` `🎯` `🚀` `⚡` `💪` `👏` `🔝` `🎊` `🏅`

Setiap yang ketemu:
- Kalau di toast/celebration → hapus, ganti dengan teks tenang
- Kalau di icon stage/label → ganti dengan emoji tenang (🌿 🕌 🌙 📖 ✦) atau SVG icon
- **Pengecualian:** 🌙 di greeting OK (bulan sabit, spiritual), 🌿 untuk growth stage OK

### 4.2 — Warna Non-Sanctuary

Cari di seluruh `src/`:
- `red-400` `red-500` `red-600` (Tailwind default red) → ganti ke `rose-600` (yang sudah di-set jadi muted purple-rose)
- `green-` `emerald-` `mint` pure → ganti ke `gold` atau hapus
- `blue-` `sky-` `cyan-` → ganti ke `violet`
- `yellow-` `amber-` `orange-` → ganti ke `gold`

### 4.3 — Sisa Brand "Madad"

Cari `MADAD` / `Madad` (case-sensitive) di copy/text:
- Semua ganti ke `Talqih`
- **KECUALI** localStorage keys (`madad_notes`, dll) — biarkan untuk backward compat
- File comment header (`/* Madad, ... */`) → update ke `/* Talqih, ... */` (opsional, kosmetik)

### 4.4 — Copy Anti-Pattern

Baca semua copy UI, cari:
- Kata "challenge", "mission", "goal", "target" dalam konteks gamifikasi → ganti bahasa yang lebih tenang
- Celebration berlebihan ("Selamat!", "Hebat!", "Keren!") → ganti dengan apresiasi tenang ("Tercatat", "Barakallahu fik")

## BAGIAN 5 — LOADING STATE (Polish)

### Fix 5.1 — Loading Placeholder Saat Babel Compile

**Lokasi:** `index.html`, dalam `<div id="root">`

**Masalah:** Babel standalone compile semua jsx di browser — di koneksi lambat (Mesir!), user lihat blank screen 3-5 detik.

**Fix:** Isi `<div id="root">` dengan loading placeholder yang akan auto-replaced saat React mount:

```html
<div id="root">
  <div style="min-height:100vh;display:flex;flex-direction:column;align-items:center;justify-content:center;background:#0A0514;">
    <div style="font-family:'Aref Ruqaa','Amiri',serif;font-size:2rem;color:#C9A86A;direction:rtl;opacity:0.85;margin-bottom:1rem;">
      تَلْقِيح
    </div>
    <div style="font-family:'Crimson Pro',serif;color:#9B8DBA;font-size:0.9rem;">
      Membuka Talqih...
    </div>
    <div style="margin-top:1.5rem;width:32px;height:32px;border:2px solid rgba(124,77,255,0.2);border-top-color:#7C4DFF;border-radius:50%;animation:talqih-spin 0.8s linear infinite;"></div>
  </div>
</div>
<style>
  @keyframes talqih-spin { to { transform: rotate(360deg); } }
</style>
```

React akan auto-replace konten `#root` saat mount. Loading ini cuma muncul sebelum app siap.
