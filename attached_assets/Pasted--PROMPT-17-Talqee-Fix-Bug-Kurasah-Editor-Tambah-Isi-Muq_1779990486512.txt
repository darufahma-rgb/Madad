# BAGIAN 1 — FIX BUG KURASAH EDITOR (PRIORITAS)

## Diagnosis

**Gejala:** Ketik 1 huruf di body/title editor → kursor hilang / tidak bisa lanjut ngetik.

**Root cause:** `noteId` dan `isNew` di-derive ulang setiap render dari `path`:

```js
// MASALAH — line 18-20, ini jalan SETIAP render:
const params = new URLSearchParams(path.includes("?") ? path.split("?")[1] : "");
const noteId = params.get("id");
const isNew = path === "/kurasah/new" || path.startsWith("/kurasah/new");
```

Ketik huruf → `setBody()` → re-render → `noteId` & `isNew` di-compute ulang →
`useEffect([noteId, isNew])` fire lagi → `setBody(found.body)` atau `setBody("")` →
**body ter-reset**, kursor hilang.

## Fix di `src/pages/kurasah-editor.jsx`

### STEP 1 — Memoize noteId dan isNew

Ganti baris 18-20:

```js
// HAPUS:
const params = new URLSearchParams(path.includes("?") ? path.split("?")[1] : "");
const noteId = params.get("id");
const isNew = path === "/kurasah/new" || path.startsWith("/kurasah/new");
```

Ganti dengan:

```js
// FIX: compute sekali saat mount, tidak berubah saat re-render
const noteId = React.useMemo(() => {
  const params = new URLSearchParams(path.includes("?") ? path.split("?")[1] : "");
  return params.get("id");
}, []); // ← dependency array KOSONG — hanya compute saat mount

const isNew = React.useMemo(() => {
  return path === "/kurasah/new" || path.startsWith("/kurasah/new");
}, []); // ← dependency array KOSONG — hanya compute saat mount
```

### STEP 2 — Jangan setNote() di dalam persistSave

`setNote(updated)` di line 79 trigger re-render yang unnecessary. Hapus baris itu:

```js
const persistSave = React.useCallback((t, b, tg, src, n) => {
  if (!n) return;
  setSaveStatus("saving");
  const updated = { ...n, title: t, body: b, tags: tg, source: src, updatedAt: new Date().toISOString() };
  const allNotes = loadNotes();
  const exists = allNotes.find(x => x.id === n.id);
  const newList = exists
    ? allNotes.map(x => x.id === n.id ? updated : x)
    : [updated, ...allNotes];
  saveNotes(newList);
  markPresenceToday();
  // HAPUS: setNote(updated);  ← ini yang trigger re-render & reset
  setSaveStatus("saved");
}, []);
```

Kenapa aman dihapus: `note` object hanya dipakai sebagai "container" untuk ID dan metadata saat save. Body & title sudah di-state sendiri (`body`, `title`). `setNote(updated)` tidak dibutuhkan untuk UI.

### STEP 3 — Pastikan textarea tidak di-unmount saat re-render

Tambahkan `key` yang stable ke textarea supaya React tidak unmount/remount elemen:

```jsx
<textarea
  key="kurasah-body-editor"  {/* ← TAMBAH INI */}
  ref={bodyRef}
  value={body}
  onChange={e => handleBodyChange(e.target.value)}
  {/* ... rest props sama */}
/>
```

Dan untuk title input:

```jsx
<input
  key="kurasah-title-editor"  {/* ← TAMBAH INI */}
  value={title}
  onChange={e => handleTitleChange(e.target.value)}
  {/* ... rest props sama */}
/>
```

### STEP 4 — Pisahkan handleBodyChange dari closure

`handleBodyChange` saat ini closure yang capture `body`, `tags`, `source`, `note` — kalau ada stale closure, nilai lama terpakai. Fix dengan functional update:

```js
// GANTI:
const handleBodyChange = (v) => { setBody(v); scheduleAutoSave(title, v, tags, source, note); };

// JADI:
const handleBodyChange = React.useCallback((v) => {
  setBody(v);
  scheduleAutoSave(title, v, tags, source, note);
}, [title, tags, source, note, scheduleAutoSave]);
```

Sama untuk handleTitleChange:

```js
const handleTitleChange = React.useCallback((v) => {
  setTitle(v);
  scheduleAutoSave(v, body, tags, source, note);
}, [body, tags, source, note, scheduleAutoSave]);
```

### Acceptance Criteria Bug Fix

- [ ] Ketik huruf di body → **kursor tidak hilang**, bisa terus ngetik
- [ ] Ketik huruf di title → sama, tidak reset
- [ ] Auto-save tetap jalan (800ms setelah stop ngetik, status "Tersimpan")
- [ ] Refresh page → catatan yang sudah diketik tetap ada (localStorage)
- [ ] Tidak ada console error/warning
- [ ] Mobile test: tap textarea, keyboard muncul, bisa ketik panjang tanpa putus
