# MADAD — Rework Landing Page + Onboarding Khas Masisir

## Konteks

Ini adalah **MADAD** — premium AI learning sanctuary untuk Masisir (mahasiswa Al-Azhar Kairo). Brand: **MADAD**. Logo mark: **AIGYPT**. Developer: **Dar Dev**.

Sekarang kita rombak dua hal:

1. **Landing page** — sekarang terlalu banyak tulisan, harus dipangkas habis. Orang Indonesia tidak suka baca panjang di landing. Focus: **visual, ringkas, langsung ke value**.

2. **Onboarding & Profile** — saat ini field-nya generic ("Syariah", "Bahasa Arab", "Kedokteran"). Harus diubah supaya benar-benar **akurat untuk sistem Al-Azhar** dengan struktur per fakultas + tingkat + jurusan kondisional.

**design.md tetap source of truth. Sanctuary mood non-negotiable.**

---

## Filosofi Perubahan (BACA DULU)

### Untuk Landing:
- **Less is more**. Hapus semua paragraf yang lebih dari 2 baris.
- **Visual carries weight, not text**. Pakai mockup, ikon, ornamen — bukan deskripsi panjang.
- **Satu pesan utama**: "Panduan Masisir untuk belajar materi Al-Azhar dengan AI."
- Hapus penjelasan teknis di landing. Detail muncul di Tool Guide / Learning Path (member area).
- **Mobile-first thinking**. Masisir mostly buka via HP — landing harus enak di-scroll cepat di mobile.

### Untuk Onboarding:
- **Akurasi > kelengkapan**. Field harus match sistem Al-Azhar yang sebenarnya, bukan dibuat-buat.
- **Conditional logic**: pertanyaan follow-up cuma muncul kalau relevan (penjurusan baru ada di tingkat tertentu untuk fakultas tertentu).
- **Tetap singkat**: jangan jadikan onboarding form panjang. Tambah field hanya yang benar-benar mempengaruhi rekomendasi/path.

```js
      "ushuluddin": "ushuluddin",
      "kedokteran": "umum",
      "teknik":     "umum",
      "lainnya":    "umum",
    };
    profile.faculty = fieldToFacultyMap[profile.field] || "umum";
  }
  
  // Default level if missing (existing users)
  if (!profile.level) profile.level = "1";
  
  return profile;
};

// Pakai di getProfile():
const getProfile = () => {
  try {
    const raw = localStorage.getItem(STORAGE_KEYS.PROFILE);
    const profile = raw ? JSON.parse(raw) : null;
    return migrateProfile(profile);
  } catch(e) { return null; }
};
```

User existing tidak perlu re-onboarding — data lama diterjemahkan otomatis.

## Anti-Pattern Check

Sebelum bilang selesai, baca semua copy landing baru sambil tanya:

- [ ] Apakah ada paragraf yang bikin gue (sebagai pembaca) bilang "TLDR" dan skip? **Kalau iya, potong lagi.**
- [ ] Apakah CTA-nya cuma 1-2 button per section? **Lebih dari itu = bingungin.**
- [ ] Apakah ada section yang "menjelaskan apa itu AI" / "kenapa AI penting" / generic motivation? **Hapus. Masisir bukan target market yang butuh dijelasin.**
- [ ] Apakah landing sekarang bisa di-screenshot 1 mobile-screen dan masih cukup untuk represent produk? **Itu test kerampingan ideal.**

## Catatan Final

Lo bilang "orang males baca tulisan panjang". Itu insight yang valid banget untuk pasar Indonesia, apalagi Masisir yang biasanya lagi capek pulang kuliah. Tapi tahan godaan untuk **menggantikan tulisan panjang dengan visual ramai**. Sanctuary mood butuh **ruang kosong sebagai bagian dari kemewahan**.

Jadi prinsipnya: **hapus tulisan, kasih napas, jangan ganti dengan animasi ramai**.

Untuk data Al-Azhar, lo sudah punya riset detail (10 fakultas + jurusan). Yang gue masukkan ke struktur ini cuma 6 fakultas utama (yang accessible untuk wafidin). Sengaja gak overload onboarding dengan 10+ opsi — sweet spot adalah 6 opsi per pertanyaan (sesuai Hick's law: lebih dari 7 = bikin user freeze).

Mulai. Kalau ada ambiguitas, default ke versi paling lega dan paling sedikit copy.
